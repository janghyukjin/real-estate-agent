## 2. 단기 개선 (1-2주)

### 2.1 [High] 데이터 커버리지 확대 — 화성시(동탄) + 부천시오정구

### 2.3 [High] 첫 방문 전환율 개선 (온보딩 플로우)

### 2.4 [Medium] 결과 공유 기능

- URL 파라미터로 입력값 전달 (딥링크)

### 2.5 [Medium] 아파트 상세 정보 강화

## 4. 장기 비전 (1-2개월)

### 4.1 [Strategic] Streamlit → Next.js 마이그레이션
- **이유**: Streamlit의 한계가 성장 병목이 될 시점이 올 것
  - SEO 불가 (SPA, 크롤러 불가)
  - URL 라우팅 제한 (공유/딥링크 불편)
  - 커스텀 인터랙션 한계 (드래그, 지도 등)
  - 콜드 스타트 느림 (Streamlit Cloud 무료 플랜)
- **시기**: MAU 1,000+ 또는 유료 전환 시 검토
- **전략**: 백엔드 API (FastAPI) + Next.js 프론트 분리. `src/` 모듈은 그대로 재사용 가능.

### 4.2 [Strategic] 수익화 모델
| 모델 | 설명 | 난이도 | 예상 수익 |
|------|------|--------|----------|
| 프리미엄 스킬팩 | 전문가 검증 스킬 유료 판매 | 낮음 | 월 50-100만 |
| 맞춤 리포트 구독 | 주간 맞춤 분석 이메일/카톡 | 중간 | 월 100-300만 |
| 제휴 대출 비교 | 대출 상담 연결 수수료 | 높음 | 건당 5-20만 |
| 광고 (부동산 업체) | 분양/중개 배너 | 낮음 | 월 30-100만 |

- **1단계**: 무료로 사용자 확보 (현재) → 월 5,000+ DAU 목표
- **2단계**: 프리미엄 기능 분리 (맞춤 리포트, 고급 필터, 알림)
- **3단계**: 제휴 수익 (대출 비교, 중개 연결)

### 4.3 [Strategic] 사용자 획득 채널
1. **인스타그램 콘텐츠** (최우선)
   - "이번 주 급등/급락 TOP 5" 카드뉴스
   - "3억으로 살 수 있는 서울 아파트" 시리즈
   - 자동 생성 파이프라인: `reanalyze.py` → 이미지 생성 → 인스타 예약 게시
2. **부동산 커뮤니티** (부동산 갤러리, 네이버 카페)
   - 데이터 기반 분석 포스팅 → 집피티 링크
3. **SEO** (Next.js 마이그레이션 후)
   - "강남 아파트 시세", "생애최초 대출 한도" 등 키워드

### 4.4 [Tech] 인프라 개선
- **데이터 파이프라인**: JSON 파일 → SQLite 또는 DuckDB (쿼리 성능, 히스토리 관리)
- **모니터링**: Streamlit Cloud 사용량 추적, 에러 로깅 (Sentry 무료 플랜)
- **CI/CD**: GitHub Actions → pytest → Streamlit Cloud 자동 배포
- **환경 분리**: `main` (prod) + `beta` (테스트) 브랜치 → Streamlit 앱 2개

## 기존 뉴스/RAG 계획 재평가

`plan_news_rag_automation.md`의 계획 (4-7주)은 범위가 넓다. 재평가:

| Phase | 기존 계획 | 재평가 |
|-------|----------|--------|
| Phase 1: 뉴스 수집+표시 | 1-2주 | **유지** — 단, 네이버 API 키 발급 선행 필요 |
| Phase 2: RAG 연동 | 2-3주 | **보류** — 뉴스 탭 사용량 확인 후 결정. KR-SBERT + ChromaDB는 리소스 대비 효과 불확실 |
| Phase 3: n8n 자동화 | 1-2주 | **대체** — GitHub Actions로 대체 (무료, 설정 간단). n8n은 워크플로우 복잡해질 때 재검토 |

**핵심 판단**: 뉴스보다 **데이터 신선도**(일일 갱신)와 **온보딩 개선**(전환율)이 더 급하다. 뉴스는 "있으면 좋은" 기능이지 핵심 가치가 아님.
